# 음성 STT 처리 방식 조사(ing)

## 적용 방식

프론트엔드에서 브라우저 Web Speech API의 SpeechRecognition을 사용해
사용자의 음성을 실시간으로 텍스트로 변환한다.

변환된 텍스트는 기존 답변 평가 API의 answer 필드로 전송한다.

## 처리 흐름

1. 사용자가 면접 화면에서 녹음 시작 버튼을 누른다.
2. 프론트엔드에서 SpeechRecognition을 시작한다.
3. 한국어 음성을 텍스트로 변환한다.
4. 변환된 텍스트를 답변 입력창에 표시한다.
5. 사용자가 답변을 제출한다.
6. 프론트엔드는 기존 POST /api/interview/feedback API를 호출한다.
7. 백엔드는 기존 방식으로 답변을 평가하고 저장한다.

## 백엔드 변경 여부

현재 단계에서는 별도의 음성 파일 업로드 API나 STT API를 만들지 않는다.
프론트에서 음성을 텍스트로 변환한 뒤 기존 answer 필드로 전달하므로
기존 답변 평가 API를 그대로 사용할 수 있다.

## 주의사항

- SpeechRecognition을 지원하지 않는 브라우저가 있으므로 지원 여부를 검사해야 한다.
- 한국어 인식을 위해 lang 값을 ko-KR로 지정한다.
- 마이크 권한 거부에 대한 오류 처리가 필요하다.
- 인식된 문장을 사용자가 제출 전에 수정할 수 있도록 한다.
